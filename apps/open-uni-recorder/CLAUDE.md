# Open University Summarizer — Claude Instructions

## Tech stack
- Node.js ESM (`"type": "module"`) — all files use `import`/`export`, no `require()`
- Express 4 (`server.js`) — single process, no build step
- Playwright (Chromium) — headless browser for OPAL SSO login
- ffmpeg (system dep, not npm) — HLS download and audio chunking; must be on PATH
- AI backends: Groq Whisper (transcription), Gemini / Claude / Groq / Ollama (summarization)
- `@anthropic-ai/sdk ^0.91`, `@google/generative-ai ^0.21`, `groq-sdk ^1.1`

## Running
```
npm start          # production: node server.js
npm run dev        # node --watch server.js (auto-restart)
```
No tests. No build. No transpilation. Just `node server.js`.

## Env vars (see `.env.example`)
Required: `OPENU_USERNAME`, `OPENU_PASSWORD`, `OPENU_ID`, `GROQ_API_KEY`
Optional: `GEMINI_API_KEY`, `ANTHROPIC_API_KEY`, `OLLAMA_HOST`
Defaults: `WHISPER_BACKEND=groq-whisper`, `SUMMARIZE_BACKEND=gemini`, `WHISPER_CONCURRENCY=2`
`config.js` is the single source for env reads — don't read `process.env` elsewhere except in backend files.

## Data layout
- `recorder-db/` (sibling of project dir) — all data. DB at `recorder-db/recorder.db`, files at `recorder-db/classes/<classId>/lectures/<lectureId>/`
- `meta.json` files are written alongside every DB write as backup for reload-from-disk
- `tmp/chunk_NNN.wav` — 10-min audio segments, deleted after transcription
- See `docs/sqlite-storage.md` for schema and migration details
- See `docs/auto-pipeline.md` for pipeline feature (detect, queue, cron) — that doc is authoritative

## Architecture
Pipeline: `extractVideoUrl()` → `downloadAndTranscribe()` / `downloadToMp3()` + `transcribeFromMp3()` → `mergeSummaries()`

All long-running routes stream SSE: `data: ${JSON.stringify({type, ...})}\n\n`. Types: `progress`, `token`, `done`, `error`.

Three-tier cache in `/api/summarize` (server.js:186–248):
1. Transcript cached → skip straight to summarization
2. MP3 cached → skip login/download, just split+transcribe
3. Nothing cached → full pipeline (login, download, transcribe, summarize)

Concurrent transcription: `download.js` polls for sealed `chunk_NNN.wav` files every 2s while ffmpeg is still running, dispatching transcription jobs immediately (controlled by `WHISPER_CONCURRENCY`).

Summarizer backends all export the same interface: `summarizeChunk(text)` and `mergeSummaries(chunks, onProgress, onToken)`. `lib/summarize.js` dynamically imports the right backend. `summarizeChunk` is a passthrough in Gemini/Claude (full transcript sent in one call); Groq actually chunks.

Whisper auto-fallback: groq-whisper 429 → local whisper-cpp (see `lib/transcribe.js`).

## Routes
- `GET /classes`, `/classes/:classId`, `/classes/:classId/lectures/:lectureId` → static HTML
- `POST /api/classes/:classId/lectures/:lectureId/transcribe` → SSE (login + download + transcribe)
- `POST /api/classes/:classId/lectures/:lectureId/summarize` → SSE (reads existing transcript.txt)
- `POST /api/summarize` → SSE, legacy URL-based flow
- `GET /api/health/{groq,gemini,claude,ollama}` → connectivity checks for settings page

## Conventions
- `__dirname` reconstructed everywhere via `fileURLToPath(import.meta.url)`
- Progress callbacks: `onProgress(msg: string)`, token callbacks: `onToken(token: string)`
- Hebrew strings in UI-facing progress messages; English in `console.log`
- IDs: `{slug}-{Date.now()}` (e.g. `item-1778344225872`), generated by `makeId()` in `storage.js`
- Validate at route boundaries; trust internal module contracts

## Non-obvious gotchas
- ffmpeg must be installed system-wide (`brew install ffmpeg` on mac)
- `WHISPER_CONCURRENCY=2` is the Groq free-tier safe default — raising it causes 429s
- `DELETE_MP3_AFTER_TRANSCRIBE=true` removes audio after transcript to save disk
- Debug mode: `DEBUG=true` dumps page HTML to `/tmp/openu-debug.html` and saves screenshot to `tmp/debug-screenshot.png` on extraction failure
- `recorder-db/` is the sole data store — `saved/` and `data/classes/` no longer exist
